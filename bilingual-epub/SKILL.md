# EPUB 中英對照版製作指引

## 初始化

`epub` 目錄是電子書解壓後的根目錄。若該目錄不存在，嘗試從根目錄找到 epub 原檔並解壓縮；找不到則回報錯誤並結束。

---

## 雙語段落結構

英文段落之後**緊接**一個中文段落。中文段落使用 `translated` class，並沿用英文段落的排版 class：

```html
<p class="indent">英文原文段落。</p>
<p class="translated indent">中文譯文段落。</p>
```

若 xhtml 檔案尚未引用共用樣式表，在 `<head>` 中加入：
```html
<link href="Styles/bilingual.css" rel="stylesheet" type="text/css" />
```

`bilingual.css` 內容：
```css
#sbo-rt-content .translated {
  margin-top: .15em;
  margin-bottom: .65em;
  color: #0f3d2e;
  font-family: 'Microsoft JhengHei', 'Noto Sans CJK TC', 'PingFang TC', 'PMingLiU', sans-serif;
  line-height: 1.65;
}

#sbo-rt-content .translated-missing {
  color: #9f1239;
}
```

### 保留內文樣式

英文段落中的 `<strong>`、`<em>`、`<code>` 等樣式元素，在中文段落中要對應保留在對應詞彙上。即使中文結構和英文不同，樣式位置也要反映語意，而非對齊原文位置：

```html
<p class="indent">This is an <strong>important</strong> point.</p>
<p class="translated indent">這點很<strong>重要</strong>。</p>
```

## 跳過雙語的條件

以下內容**不加**中文對照：
- 非正文內容（目錄、版權頁等）
- 大小標題
- 圖表說明標題（若判定為句子而非標題則加）
- 表格內文字數不多且結構簡單時

其他情況由你根據書籍內容自主判定。

## 特殊元素：註腳

英文註腳的 `id` 和 `href` 屬性，中文版要加 `-tw` 後綴，讓兩組連結各自獨立運作：

```html
<!-- 英文 -->
<p class="footnote"><a id="ch02fn1" href="ch02.xhtml#ch02fn1a">1</a>.</p>
<sup><a id="ch02fn1a" href="ch02.xhtml#ch02fn1">1</a> This is Detail.</sup>

<!-- 中文 -->
<p class="translated footnote"><a id="ch02fn1-tw" href="ch02.xhtml#ch02fn1a-tw">1</a>.</p>
<sup><a id="ch02fn1a-tw" href="ch02.xhtml#ch02fn1-tw">1</a> 這是細節。</sup>
```

## 中文表達風格

目標是讀起來像台灣人直接用中文寫的，而不是從英文翻過來的。

**核心原則：**
- 先理解英文意思，再用中文從頭表達，不逐句對照
- 句子可長可短，重點是層次清楚、停頓自然、一路讀下去不拗口
- 不適合用中文表達的英文詞彙（技術詞彙、人名等）直接保留
- 永遠使用台灣慣用語與翻譯名詞

**去翻譯腔的具體手法：**
- **語態轉換**：英文常寫被動，中文多半主動更自然；若直譯顯得僵硬，就改寫語態
- **分句 / 合句**：判準是中文節奏是否連貫，不是字數；避免為了對齊英文句號硬切或硬拖
- **正說反譯**：若中文用正說、反說或雙重否定更順，就改寫，不要複製英文句型表面
- **正式度控制**：技術書正文維持清楚、穩定、書面，但不能寫成公文腔

**直譯警示徵兆：**
- 被動句過多，讀起來像翻譯文件
- 每句長度都和英文差不多，沒有依中文節奏調整
- 轉折、因果、補充全靠連接詞硬串
- 句子邏輯沒錯，但中文讀者不會這樣說

**標點符號限制：** 除非原文已使用，否則禁止使用冒號（`：`）、分號（`；`）、破折號（`——`）。

## 工作流程

### 新增中文到章節

1. 讀英文段落，判斷是否需要跳過（見上方「跳過雙語的條件」）
2. 理解語意 → 判斷語態、分句合句、正反互換的需要 → 用中文重新表達
3. 插入緊接在英文段後
4. 每段完成後對照中英文，確認語意正確、節奏自然、正式度合宜

### 修改現有中文

1. 逐段讀取中文段落，參照英文原文理解意思
2. 檢查被動句過多、分句失衡、正反表達不自然、正式度不對等問題
3. 按照上述風格原則重寫，可保留原本段落的優良部分

### 輸出打包

將 `scripts/pack.bat` 複製到呼叫端工作目錄，執行後產出打包結果。

## 驗收

章節完成後逐項驗收，有問題則修改後再次驗收，直到全部通過：

1. **語意**：中英文對照，確認意思沒有錯誤或遺漏
2. **文風**：符合「白話精煉」要求，不像翻譯腔
3. **順讀**：只讀中文段落時，應像原生中文書面說明，不讓讀者感到英文句法殘影
4. **正式度**：正文、側欄、註腳、表註符合各自應有的書面正式度；不口語過頭，也不板成公文腔
